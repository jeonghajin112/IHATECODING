## Acceptance 기준 대조

- 모든 Rust ready 측정값은 `C# + 500 ms`와 `C# × 1.20` 중 큰 값 이하라는
  기준을 충족했다.
- 20-pane steady tree working set 평균은 Rust 2,113.5 MiB, C# 2,080.9 MiB로
  Rust가 32.6 MiB 높았다. 20-pane peak tree 평균도 Rust가 약 32.7 MiB
  높았지만, 두 값 모두 `PHASE6_ACCEPTANCE.md`의 허용 예산 안이었다.
- 가장 긴 정상 종료는 1,492 ms로 8초 제한 안이었다. 모든 실행에서 종료 후
  남은 검증 대상 프로세스 수는 0이었다.
- 따라서 이 세 보고서는 Phase 6의 자동 비교·성능·종료 게이트를 통과했다.

## Legacy 자동 가져오기 smoke

`phase6-package-import-smoke.json`의 Rust 8-pane 실행은 `pass`였고
`legacyImportVerified`가 `true`였다. 시작 1,103 ms, ready 1,836 ms, 종료
974 ms였으며 종료 후 남은 프로세스 수는 0이었다. 이 실행도 아래와 동일한
최종 package Rust 바이너리를 사용했다.

## 측정 바이너리 SHA-256

- Rust: `ad312a905b35dd5fdd9e197239db1d636de26b87261b33166fac2ca42b5f25b2`
- C#: `55dc9cfe12b3252c69a291a8491a71f3625d2266ce64b1273db19d0aaac05b9f`

## NSIS QA 패키지

- 파일: `IHATECODING_1.0.0_x64-setup.exe`
- 크기: 2,473,611 bytes (2.36 MiB)
- SHA-256: `008b01172ad1dd5b7b67287d4f48d4b7cce48c7431c9b683ff6e513964d5668f`
- Authenticode 상태: `NotSigned`

이 설치 파일은 서명되지 않은 QA 패키지다. 위 비교 및 import smoke에 사용한
release 실행 파일의 SHA-256은 Rust 항목과 일치하지만, `NotSigned` 패키지를
서명된 production 배포본으로 간주하지 않는다.

## 로컬 개발 컷오버

작업 폴더의 기본 `IHATECODING.exe`는 위 Rust SHA-256과 일치하는 후보로
전환했다. 전환 전에 기존 C# 실행 파일을 별도 롤백 파일로 복사하고 원본과
SHA-256이 일치하는지 확인했다. 전환 뒤 `phase6-post-cutover-smoke.json`으로
1/8/20-pane을 다시 실행했으며 모두 `pass`, 종료 후 남은 프로세스는 0이었다.

이 전환은 명시적으로 허용한 로컬 미서명 개발 컷오버다. 서명된 production
배포 승인이나 아래 별도 게이트를 대신하지 않는다.

## 남은 별도 게이트

이 문서는 JSON으로 검증된 자동 비교와 legacy import 결과만 판정한다. IME,
